


# StyleFlow Marketing Hub & Clothing Store System - שלב ה'

ברוכים הבאים לדו"ח הסיכום והוראות ההפעלה של הפרויקט. קובץ זה מציג את כלל העבודה שבוצעה בשלב ה', הכוללת בניית אפליקציית Full-Stack מלאה המחוברת לבסיס הנתונים של המערכת.

---

## 1. הסבר על דרך העבודה והארכיטקטורה

הפרויקט בנוי בארכיטקטורת **Client-Server (Full-Stack)** ומבוסס על שלוש שכבות מרכזיות:

1. **שכבת בסיס הנתונים (Database Layer):** שימוש ב-**PostgreSQL** המארח את כלל הטבלאות, הקשרים, והלוגיקה העסקית המתקדמת שפותחה בשלבים הקודמים (פונקציות, פרוצדורות וטריגרים).
   
2. **שכבת השרת (Backend Layer):** שרת המבוסס על **Node.js** בשילוב **TypeScript** ו-**Express**. השרת חושף נקודות קצה (API Endpoints) ומבצע שאילתות דינמיות מול ה-Database באמצעות חבילת `pg` (Connection Pool). השרת מנהל בצורה מאובטחת את פעולות ה-CRUD ומפעיל את תתי-התוכניות של ה-SQL.

3. **שכבת הממשק (Frontend Layer):** אפליקציית **React** מודרנית המעוצבת כ-Admin Panel דינמי. הממשק מאפשר למשתמש קצה לנהל את המידע בצורה גרפית ללא צורך בכתיבת קוד SQL ידני.

---

## 2. הטכנולוגיות והכלים בהם השתמשנו

במהלך פיתוח האפליקציה השתמשנו בכלים ובטכנולוגיות הבאות:

* **שפות פיתוח:** TypeScript, JavaScript, SQL (PL/pgSQL).
* **סביבת הרצה ושרת:** Node.js, Express, TSX (לעבודה חלקה עם TypeScript ב-Backend).
* **ספריית ממשק משתמש:** React.js בשילוב Vite כמנוע הרצה מהיר.
* **עיצוב ורספונסיביות:** Tailwind CSS בשילוב ספריית האייקונים Lucide React.
* **ניהול בסיס הנתונים:** pgAdmin 4 ו-PostgreSQL המנוהל מקומית או בענן.
* **כלי פיתוח ובדיקות:** Git & GitHub, דפדפן וכלים מובנים (DevTools) לבדיקת רשת.

---

## 3. הוראות הפעלה והרצה של האפליקציה

כדי להריץ את האפליקציה בצורה מקומית, יש לעקוב אחר השלבים הבאים:

### א. הכנת שרת בסיס הנתונים (PostgreSQL)
1. יש לוודא כי ה-Database פועל וכי הרצתם בו את כל הסקריפטים של הטבלאות והנתונים.
2. יש לוודא כי הפונקציה `fn_calculate_campaign_efficiency` והפרוצדורה `sp_adjust_campaign_budgets` מותקנות וקיימות ב-DB.
3. יש ליצור קובץ בשם `.env` בתיקיית השורש של הפרויקט ולהזין בו את פרטי החיבור:
   ```env
   DB_USER=your_username
   DB_HOST=localhost
   DB_DATABASE=your_database_name
   DB_PASSWORD=your_password
   DB_PORT=5432

```

### ב. הרצת השרת האחורי (Backend)

1. פתחו את הטרמינל בתיקיית השרת.
2. התקינו את חבילות הקוד על ידי הפקודה:
```bash
npm install

```


3. הפעילו את השרת על פורט 8080 באמצעות הפקודה:
```bash
PORT=8080 npx tsx server.ts

```


*לאחר ההפעלה, תופיע בטרמינל הודעה המאשרת חיבור מוצלח לבסיס הנתונים.*

### ג. הרצת הממשק הקדמי (Frontend)

1. פתחו חלון טרמינל חדש בתיקיית ה-Frontend.
2. התקינו את חבילות הקוד (אם טרם הותקנו):
```bash
npm install

```


3. הרצו את פקודת הפיתוח:
```bash
npm run dev

```


4. פתחו את הדפדפן בכתובת המופיעה בטרמינל (לרוב `http://localhost:5173`).

## 4. פירוט הפונקציונליות הקיימת במערכת

ממשק המשתמש מחולק בצורה ברורה לשני אזורי עבודה מרכזיים:

### 1. מסכי ניהול מידע (CRUD Workspace)

* **תצוגת טבלאות דינמית:** המערכת תומכת ב-10 טבלאות שונות מתוך בסיס הנתונים (כולל קמפיינים, לקוחות, מוצרים, ספקים ומלאי). המערכת מבצעת שאילתות `JOIN` מורכבות כדי להציג שמות ברורים במקום קודים מספריים.
* **עדכון נתונים (Update):** לחיצה על כפתור העריכה (`📝`) פותחת מודל המאפשר לשנות ערכים (כגון תקציב). השרת מנקה ומסנן שדות תצוגה ומבצע בקשת `PUT` פיזית מול עמודות ה-DB.
* **מחיקת נתונים (Delete):** לחיצה על כפתור המחיקה מסירה את הרשומה ישירות מבסיס הנתונים על בסיס המפתח הראשי שלה (`DELETE`).

### 2. הפעלת תתי-תוכניות (Stored Subprograms)

* **פונקציה (F1: Campaign Efficiency):** מאפשרת להזין מזהה קמפיין, קוראת לפונקציה השמורה ב-DB ומציגה על המסך את מדד היעילות המשוקלל שחושב בזמן אמת.
* **פרוצדורה (P2: Adjust Budgets):** מאפשרת להזין מזהה מנהל ותקרת תקציב, ומבצעת קריאת `CALL` לפרוצדורה שמבצעת אופטימיזציה, הגדלה או הקטנה של תקציבי הקמפיינים ישירות בתוך ה-Database עם חיווי הצלחה גרפי למשתמש.
